Mars 2004 à septembre 2004 : Architecte d'une solution de migration progressive de l'offre du MIPIH écrite en Natstar vers J2EE.

Toute l'offre Convergence du MIPIH est développée en Natstar. Cet AGL génère du C portable sous Windows et sous Unix qu'il compile sous formes de bibliothèques dynamiques (DLL sous windows). J'ai réalisé pour le MIPIH un prototype destiné à prouver que la réutilisation du code natif depuis Java était possible. Mon travail a donc consisté en l'étude des différentes possibilités techniques d'interfaçage du Java avec ces DLL.

Mon choix s'est porté sur l'utilisation d'un outil éprouvé pour appeler le natif depuis Java et démontrer ainsi la faisabilité du principe : JNI.

Une fois cette question purement technique d'interfaçage résolue, j'ai conçu l'architecture du logiciel pour permettre à terme de remplacer de façon simple et transparente l'appel au code natif (les DLL) par du pur code Java. J'ai utilisé le design pattern DAO pour ce faire.

Environnement : Eclipse, Java, C, JNI, Tomcat, JSP/Servlets.

2004 - 2e année d'école d'ingénieurs
Moteur de résolution de problèmes par algorithmes génétiques (Java)

Il existe des problèmes qui ne possèdent pas de solution en "temps humain". Le célèbre problème du voyageur de commerce par exemple, où on souhaite qu'un représentant passe par un certain nombre de ville dans son trajet et que ce voyage soit optimisé, ne peut plus se calculer par ordinateur en un temps raisonnable dès que le nombre de villes dépasse une valeur assez petite (de l'ordre d'une vingtaine).

On utilise la théorie de sélection naturelle (Darwin) pour trouver des solutions de façon aléatoire et les améliorer par paliers successifs de "mutation", "croisement" et "reproduction" des meilleures solutions.

Ce type d'algorithme permet de trouver des solutions acceptables en un temps raisonnable.

Implémentation de l'algorithme du Spanning Tree - Simulation d'Ethernet au dessus d'UDP (Java)

Projet appelé EoU : Ethernet over UDP. Nous avons utilisé le multicast UDP pour créer l'équivalent de ce qui se passe au niveau Ethernet entre des machines. Une fois cette couche écrite, écriture du Spanning Tree selon la spécification de l'IETF afin supprimer les boucles entre les switchs virtuels.
